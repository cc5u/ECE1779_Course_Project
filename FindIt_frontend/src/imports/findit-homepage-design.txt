Create a modern, clean home page UI for a cloud-native lost-and-found web application called "FindIt". The main goal of the page is to allow users to view lost item reports on an interactive map and quickly report a new lost item.

Layout requirements:

Overall layout:

* Desktop-first design (1440px width).
* Minimal, modern, professional aesthetic similar to Google Maps or Airbnb.
* Use a light theme with white background, subtle gray borders, and blue as the primary accent color.
* Use a clean sans-serif font (Inter, Roboto, or similar).

Top navigation bar:

* Fixed header at the top.
* Left side: logo icon and text "FindIt".
* Right side: buttons "Report Lost Item" (primary button, blue), "Login", and "Sign Up".
* Navigation bar should have subtle shadow and white background.

Main content area:

* The main area should be dominated by an interactive map container (placeholder rectangle labeled "Map View").
* The map should take about 70–75% of the page width, aligned to the left.
* The map should show example circular pins indicating lost item locations (use simple colored circles with radius).

Right sidebar panel:

* A vertical sidebar on the right (about 25–30% width).
* Title at top: "Recent Lost Reports".
* Display 3 example lost item cards stacked vertically.
* Each card should contain:

  * small circular thumbnail placeholder image
  * item name (e.g., "Lost Black Wallet")
  * location (e.g., "Downtown Toronto")
  * time (e.g., "2 hours ago")
  * status badge (e.g., "Lost", "Possibly Found")

Floating action button:

* Add a floating circular "+" button on the bottom-right corner of the map labeled "Report".
* Use blue primary color.
* Should look modern and clickable.

Hero message overlay on map (optional subtle overlay at top-left of map):

* Text: "Help people find their lost belongings"
* Subtext: "Report lost items or help others by marking found items."

Design style:

* Clean spacing using an 8px grid system.
* Rounded corners (8–12px radius).
* Soft shadows for cards.
* Use simple placeholder shapes for map pins, avatars, and images.
* No need to implement real map functionality—just realistic visual placeholders.

Deliverables:

* One complete desktop home page frame.
* Well-organized auto layout.
* Reusable components for navbar, report card, and buttons.
